Two patterns you’ll use often in Angular: async pipe (subscribe in the template, no cleanup code) and BehaviorSubject in a service (one place holds the value, components read it via an observable).

angular-rxjs.ts 
{{ codeAngular }}

1. async pipe — subscribe in the template

The template has {{ count$ | async }}. Angular subscribes when the component is shown and unsubscribes when it’s destroyed. You never write subscribe() or unsubscribe().

Current value (from count$ | async)
{{ state.count$ | async }}

Try: click Increment. The number updates because the observable emitted a new value.

2. BehaviorSubject in a service — shared state

A service holds a BehaviorSubject. Components get a stream via state$ = this.stateSubject.asObservable() and display it with state$ | async. When something calls next(newValue), every template using state$ | async updates.

What the template shows (state$ | async)
{{ state.state$ | async }}

This is the same text you typed. The service is the single source of truth; this component just subscribes and displays it.
