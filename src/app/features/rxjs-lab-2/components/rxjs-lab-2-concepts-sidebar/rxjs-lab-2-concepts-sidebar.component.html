map()

Transforms each emitted value. Essential for shaping data before it reaches the UI.

filter() / distinctUntilChanged()

filter() keeps only values that pass a condition. distinctUntilChanged() emits only when the value is different from the previous one, avoiding redundant work and duplicate API calls.

debounceTime()

Waits until the source has not emitted for a given time, then emits the latest value. Crucial for search: wait for the user to stop typing before firing a request.

switchMap()

Maps to an inner observable and cancels the previous inner subscription when the outer emits again. Use for search: each new query cancels the previous request so only the latest result matters.

} @if (activeTab() === 'catch-error') {

catchError()

Handles errors inside the pipeline. Return a fallback observable (e.g. of(value)) so the stream continues instead of breaking. Enables retry or default data.

} @if (activeTab() === 'take') {

take(1)

Takes the first emission then completes and unsubscribes. Ideal for one-off data (e.g. a single config or route param).

takeUntil()

Completes when a notifier observable emits (e.g. a destroy$ Subject in ngOnDestroy). Standard pattern to avoid memory leaks: observable$.pipe(takeUntil(destroy$)).

} @if (activeTab() === 'combine') {

combineLatest()

Combines multiple observables and emits an array of the latest value from each whenever any of them emits. Useful for dashboards or UIs that depend on several changing sources (e.g. filters).

} @if (activeTab() === 'angular') {

async pipe

Subscribes to an observable in the template and unsubscribes when the component is destroyed. Best practice for subscriptions in templates — no manual subscribe() or ngOnDestroy cleanup.

Services & BehaviorSubject

Use BehaviorSubject in a singleton service to hold app-wide or feature state. Expose asObservable() for read-only streams; call next() to update. Fits Angular DI and works with the async pipe.
