TrackBy Lab

See how track in @for affects DOM reuse. Without a stable identity, Angular destroys and recreates list items on every change. With track item.id, it reuses existing DOM nodes when items move or update.

@if (lastAction(); as action) {
@if (action === 'add') {

Adding an item creates one new list entry. Both lists create one new component instance.

} @if (action === 'reorder') {

Without trackBy (track item): Reordering changes which object is at each position. Same object refs in a new order — Angular reuses views. With trackBy (track item.id): Same — Angular moves existing DOM nodes. Both reuse; the difference shows when you Refresh.

} @if (action === 'update') {

Without trackBy: The first item gets a new object reference. With track item, that's a new identity — one new creation. With trackBy: Same item.id — Angular reuses the view and updates the binding. No new creation.

} @if (action === 'refresh') {

Without trackBy (track item): The array is replaced with new objects (same data, new refs). Every item has a new identity — Angular destroys all and creates new ones. With trackBy (track item.id): Same ids — Angular reuses all views and updates bindings. No new creations.
