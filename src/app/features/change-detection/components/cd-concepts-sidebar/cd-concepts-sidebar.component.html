Zone.js

Default strategy relies on Zone.js. Zone patches browser async APIs (e.g. setTimeout, addEventListener, fetch). After any of these run, Zone notifies Angular and a change detection cycle runs. Your Default component is then checked on every cycle — no extra code needed.

ChangeDetectionStrategy

Each component has a strategy that tells Angular when it may check that component for changes. Default means: check this component on every change detection run.

Default strategy

With ChangeDetectionStrategy.Default, the component is visited on every CD cycle. Simple and predictable, but in a large tree it can mean a lot of work. Use it when the tree is small or you want straightforward behavior.

Zone.js

With Zone.js, any async action triggers a CD cycle. OnPush does not change when the cycle runs — it only controls whether this component is checked. So Zone still triggers the run; OnPush decides “skip this component unless inputs/events/signals say otherwise.”

Zoneless

OnPush works in zoneless mode too. Without Zone, CD runs only when something Angular tracks changes (e.g. signal update, template event). OnPush still applies: the component is only checked when an @Input() reference changes, an event from the component fires, or a signal read in the template changes.

The CD tree walk

When Angular runs change detection it starts at the root and visits every component. Each node is checked or skipped based on its strategy and whether it was marked dirty.

Default — always checked

ChangeDetectionStrategy.Default components are visited on every CD pass triggered by Zone.js. They are never skipped, regardless of whether inputs or state changed.

OnPush — skipped unless dirty

OnPush components are skipped unless Angular marked them dirty. An @Input reference change, a component event, or an async pipe emission will mark it dirty for the next CD pass.

Signal — targeted update

A signal write bypasses the tree walk entirely. Angular schedules only the specific views that read the changed signal — no CD pass, no zone tick needed.

Zoneless

Signals are the main way to drive updates in zoneless mode. When a signal changes, Angular schedules only the views that read it — no Zone needed. This lab runs with Zone, but the same signal code works in zoneless apps.

Signals

Signals are reactive values. When you read a signal in a template and it changes, Angular updates only the affected view instead of walking the whole tree. Components that use signals typically use OnPush. Signals work the same with or without Zone.js.

OnPush with signals

Signal-based components usually use ChangeDetectionStrategy.OnPush. The signal update is what marks the view dirty; OnPush just keeps the rest of the tree from being checked unnecessarily.
