Concepts

Core concepts

Observables are the building block for async and event streams. They are lazy (start only when subscribed) and cancellable with unsubscribe(). Used by HttpClient, ActivatedRoute, FormControl.valueChanges.

Subjects multicast values to many observers. BehaviorSubject holds the current value and replays it to new subscribers — common for app-wide state in services.

Operators are pure functions used inside .pipe() to transform, filter, and combine streams declaratively.

mergeMap

Each outer value starts an inner observable. All inner observables run in parallel; every completion is emitted. Good for "each click starts a request, show all results."

switchMap

Each new outer value cancels the previous inner observable. Only the latest inner result is emitted. Good for "only care about the latest search/request."

} @if (activeTab() === 'error-retry') {

retry

retry(n) resubscribes to the source up to n times when it errors. The source runs again from the start. If it keeps erroring, the subscription errors after retries.

Error handling

Use catchError to return a fallback stream, or retry to try again. The log shows next, error, and when a retry happens.

} @if (activeTab() === 'cold-hot') {

Cold

A cold observable runs its producer for each subscriber. of(1,2,3) is cold: each subscriber gets 1, 2, 3 from an independent run.

Hot

A hot observable shares one run among subscribers. share() turns a cold observable into hot by multicasting. Late subscribers may miss earlier emissions.
