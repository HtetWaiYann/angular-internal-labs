Guards run after routes are recognized and before the component activates. canActivate blocks forward navigation; canDeactivate blocks leaving. Return true to allow, false to block, or a UrlTree to redirect. The router wraps the entire guard phase in GuardsCheckStart / GuardsCheckEnd.

demo.guard.ts
export const demoGuard: CanActivateFn = (route, state) => {
  // Sync: return true | false | UrlTree
  // Async: return Observable<boolean> | Promise<boolean>
  return true;
};

// Attach to a route:
{ path: 'guarded', canActivate: [demoGuard], loadComponent: ... }

} @if (activeTab() === 'resolvers') {

Resolvers run after all guards pass and before the component activates. The resolved value is available synchronously via ActivatedRouteSnapshot.data the moment the component's constructor runs — no loading state needed. The router wraps resolver execution in ResolveStart / ResolveEnd.

demo.resolver.ts
export const demoResolver: ResolveFn<DemoData> = (route, state) => {
  // Return sync data, Observable, or Promise:
  return httpClient.get<DemoData>('/api/demo');
};

// Attach to a route:
{ path: 'resolved', resolve: { demoData: demoResolver }, loadComponent: ... }

// Access in component:
constructor(private route: ActivatedRoute) {
  const data = route.snapshot.data['demoData'];
}

} @if (activeTab() === 'lazy') {

Lazy-loaded routes use loadComponent or loadChildren — the JS chunk is fetched on-demand the first time the route activates. RouteConfigLoadStart fires when the chunk is requested; RouteConfigLoadEnd fires when it is ready. Subsequent visits reuse the cached module and fire no extra events.

app.routes.ts
// Single lazy component:
{
  path: 'lazy',
  loadComponent: () =>
    import('./lazy.component').then(m => m.LazyComponent)
}

// Lazy route module (multiple routes in one chunk):
{
  path: 'lazy',
  loadChildren: () =>
    import('./lazy.routes').then(m => m.LAZY_ROUTES)
}
ℹ The first visit to /router/lazy loads the chunk and fires the events below. Revisiting reuses the cached module — reload the page to see the event again.
