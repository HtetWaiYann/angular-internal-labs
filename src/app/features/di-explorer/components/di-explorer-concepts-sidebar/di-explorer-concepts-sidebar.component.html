What is an injector?

An injector is a container that holds providers and can create instances. Angular builds a tree of injectors: NullInjector → Platform → Root → … → Component.

Lookup direction

When you call inject(MyService), Angular starts at your component's injector and walks up to the parent until it finds a provider. If it reaches NullInjector without finding one, you get NullInjectorError.

Child inherits parent

A component can inject anything its ancestors provide. A root provider is visible to every component in the app.

providedIn: "root"

Registers the service in the RootInjector. One instance for the entire app. Tree-shakeable: if no one injects it, it won't be in the bundle.

providers: [...]

Registers in the component's injector. A new instance per component (unless you use a different scope). Visible only to that component and its descendants.

Multiple instances

If you put a service in a component's providers, each instance of that component gets its own service instance. Children share the parent's instance.

Token = key

A token is what you pass to inject() — usually a class. Angular uses it as a key to find the right provider in the injector tree.

Resolution steps

  1. Start at the requesting component's injector
  2. Check if this injector has a provider for the token
  3. If not, go to parent injector and repeat
  4. If found, create instance (or return existing) and return

Singleton vs per-component

Root providers create one instance. Component providers create one per component instance. The first injection creates it; later injections reuse it.

When does injection happen?

When Angular creates a component, it runs the constructor and field initializers. inject(MyService) runs at that moment — synchronously. The service is created on first request and cached.

inject() vs constructor

inject(MyService) can be used in field initializers and inject() function. Constructor injection constructor(private s: MyService) is equivalent — both use the same injector.

Where to inject

Use inject() in: constructors, field initializers, runInjectionContext. Do not use it in: lifecycle hooks, event handlers, or async callbacks — the injection context is gone there.
